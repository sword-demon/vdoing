# Cuelang 了解和基本使用

# 简介

Cue 是一种开源数据约束语言，旨在简化涉及定义和使用数据的任务。

它是json的超集，让熟悉json的用户快速上手。

github 地址 (opens new window)

# 安装

现在是go 1.17或者更高的使用

go install cuelang.org/go/cmd/cue@v0.4.2
1

会安装到你的GOPATH/bin目录下，安装完了就可以使用cue命令

➜  cue version
cue version v0.4.2 darwin/arm64
1
2

# 基础使用

创建一个test.cue文件，如果是goland可以安装一个cue的扩展去识别该文件。

user: {
	name: "wujie",
	age: 19
}

# 简单验证

设置类型

#metadata: {
	name: string
}
1
2
3

验证生效

metadata: #metadata & {
	name: "nginx"
}
1
2
3

如果想要字符串和整型都支持

#metadata: {
	name: string | int
}
1
2
3

如果写了类型，但是你下面没写，会提示错误要你填写

# 可选属性

#metadata: {
	name: string
	namespace?: string
}
1
2
3
4

可填可不填，可以在键后面加上问号

#metadata: {
	name: string
	namespace?: string | *"default"
}
1
2
3
4

此时，如果你不填，生成的对象就没有namespace

➜ cue export test.cue -e pod --out yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
    - image: nginx:1.18-alpine
1
2
3
4
5
6
7
8
9

如果把问号去掉且你还是没填，则会给你生成一个默认的namespace对应的值为default

➜  cue export test.cue -e pod --out yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  namespace: default
spec:
  containers:
    - image: nginx:1.18-alpine

但是如果你加了问号，也自己写了对应的内容，就会显示你写的对应的内容。

还可以使用变量来代替默认值的设置

let defaultns = "default"

#metadata: {
	name: string
	namespace: string | *defaultns
}
